PeonPing includes an [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) server so any MCP-compatible AI agent can play sounds directly via tool calls â€” no hooks required.

## Key difference

**The agent chooses the sound.** Instead of automatically playing a fixed sound on every event, the agent calls `play_sound` with exactly what it wants:

- `duke_nukem/SonOfABitch` when a build fails
- `sc_kerrigan/IReadYou` when reading files
- `glados/YourEntireTeamIsDead` when tests fail

This gives the agent full creative control over audio feedback.

## Requirements

- Node.js 18+
- PeonPing installed (macOS, Linux, or WSL2)

    ```json
    {
      "mcpServers": {
        "peon-ping": {
          "command": "node",
          "args": ["/path/to/peon-ping/mcp/peon-mcp.js"]
        }
      }
    }
    ```
    
    ### Cursor
    
    Edit `.cursor/mcp.json`:
    
    ```json
    {
      "mcpServers": {
        "peon-ping": {
          "command": "node",
          "args": ["/path/to/peon-ping/mcp/peon-mcp.js"]
        }
      }
    }
    ```
    
    ### Any MCP client
    
    ```json

      "command": "node",
      "args": ["/path/to/peon-ping/mcp/peon-mcp.js"],
    ```

  </Step>

## MCP features

### Tool: play_sound

Play one or more sound effects.

**Parameters:**
- `sound` (string, optional) â€” Single sound key like `"peon/PeonReady1"`
- `sounds` (array, optional) â€” Multiple sound keys like `["peon/WorkWork", "peon/JobsDone"]`

**Examples:**

```json
// Play one sound
{
  "sound": "duke_nukem/Groovy"
}

// Play multiple sounds
{
  "sounds": ["glados/OhItsYou", "glados/Wonderful"]
}
```

The agent can call this tool whenever it wants audio feedback.

### Resource: peon-ping://catalog

Full pack catalog â€” all packs and sounds. The client reads this once, giving the model full context without repeated tool calls.

**Example output:**

```
75 packs. Use play_sound with "pack/SoundName".

duke_nukem (Duke Nukem): duke_nukem/Groovy ("Groovy"), duke_nukem/SonOfABitch ("Son of a bitch!"), ...
glados (GLaDOS): glados/OhItsYou ("Oh, it's you."), glados/YourEntireTeamIsDead ("Your entire team is dead."), ...
peon (Orc Peon): peon/PeonReady1 ("Ready to work?"), peon/WorkWork ("Work, work."), ...
```

### Resource: peon-ping://pack/{name}

Sounds in a specific pack, grouped by category.

**Example: peon-ping://pack/peon**

```
Orc Peon:
session.start:
  peon/PeonReady1 â€” "Ready to work?"
  peon/PeonReady2 â€” "Yes?"
task.complete:
  peon/WorkWork â€” "Work, work."
  peon/JobsDone â€” "Job's done!"
input.required:
  peon/SomethingNeedDoing â€” "Something need doing?"
```

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PEON_PACKS_DIR` | `~/.openpeon/packs` | Sound packs directory |
| `PEON_VOLUME` | `0.5` | Playback volume (0-1) |

Set these when configuring the MCP server:

```json
{
  "command": "node",
  "args": ["/path/to/peon-mcp.js"],
  "env": {
    "PEON_VOLUME": "0.8",
    "PEON_PACKS_DIR": "/custom/path/to/packs"
  }
}
```

## Platform support

| Platform | Audio Backend |
|----------|---------------|
| macOS | `afplay` or `peon-play` (Sound Effects device) |
| Linux | `pw-play`, `paplay`, `ffplay`, `mpv`, `play`, `aplay` (first available) |
| WSL2 | PowerShell `SoundPlayer` |

<Warning>
  Windows native support is not yet available. Use WSL2 instead.
</Warning>

## How it works

1. **Client starts** â†’ MCP server loads pack catalog from `~/.openpeon/packs/`
2. **Client reads resources** â†’ Model gets full sound catalog via `peon-ping://catalog`
3. **Agent wants audio** â†’ Calls `play_sound` with sound keys
4. **Server plays audio** â†’ Resolves pack/sound keys to file paths, queues playback
5. **Audio plays** â†’ Native audio backend plays the file asynchronously

## Debugging

### Check if the server is loaded

In Claude Desktop, check the MCP status icon. In Cursor, check `.cursor/logs/`.

### Test manually

Run the MCP server directly:

```bash
node /path/to/peon-mcp.js
```

It should start without errors. Press Ctrl+C to stop.

### Verify packs are found

```bash
ls ~/.openpeon/packs/
# Should list: peon, glados, sc_kerrigan, etc.
```

### Check Node.js version

```bash
node --version
# Should be v18.0.0 or higher
```

## Architecture

The MCP server (`peon-mcp.js`) is a standalone Node.js script that:

1. **Loads config** â€” Reads `~/.claude/hooks/peon-ping/config.json` for volume
2. **Scans packs** â€” Reads `openpeon.json` manifests from `~/.openpeon/packs/`
3. **Exposes resources** â€” Serves catalog and pack details via MCP Resources
4. **Handles tool calls** â€” Resolves sound keys to file paths, validates, and queues playback
5. **Plays audio** â€” Spawns native audio players (afplay, pw-play, etc.) asynchronously

**No state is shared** with the hook-based PeonPing installation. The MCP server is completely independent.

## Limitations

- **No desktop notifications** â€” Only audio playback
- **No mobile notifications** â€” MCP server is client-side only
- **No relay support** â€” SSH/devcontainer routing not implemented
- **Max 5 sounds per call** â€” Prevents abuse
