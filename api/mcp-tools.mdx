# MCP Tools

The PeonPing MCP server provides a single tool for playing sounds.

## play_sound

Play one or more sounds by key.

### Parameters

<ParamField path="sound" type="string">
  Sound key in format `pack/sound` (e.g., `peon/PeonReady1`). Mutually exclusive with `sounds`.
</ParamField>

<ParamField path="sounds" type="string[]">
  Array of sound keys for batch playback (max 5). Mutually exclusive with `sound`.
</ParamField>

### Sound Key Format

Sound keys use the format: `pack_name/sound_name`

- **pack_name** â€” pack directory name (e.g., `peon`, `glados`, `duke_nukem`)
- **sound_name** â€” sound file name without extension (e.g., `PeonReady1` for `sounds/PeonReady1.wav`)

Example keys:
- `peon/PeonReady1`
- `glados/OhItsYou`
- `sc_kerrigan/IReadYou`
- `duke_nukem/SonOfABitch`

### Response

<ResponseField name="content" type="array">
  Array with a single text result object
  
  <ResponseField name="type" type="string">
    Always `"text"`
  </ResponseField>
  
  <ResponseField name="text" type="string">
    Result summary (one line per sound)
  </ResponseField>
</ResponseField>

### Batch Playback

When using `sounds` array:

- **Maximum 5 sounds** â€” requests with >5 sounds are truncated
- **Sequential playback** â€” sounds play in order via queue
- **Partial success** â€” valid sounds play even if some keys are invalid

Example batch with mixed results:

```json
{
  "tool": "play_sound",
  "sounds": [
    "peon/PeonReady1",
    "invalid",
    "glados/OhItsYou"
  ]
}
```

### Sound Discovery

To discover available sounds, read the `peon-ping://catalog` resource:

```typescript
// Agent reads catalog once per session
resource_read: {
  uri: "peon-ping://catalog"
}

// Returns all packs and sounds
{
  "contents": [
    {
      "uri": "peon-ping://catalog",
      "text": "165 packs. Use play_sound with \"pack/SoundName\".\n\npeon (Orc Peon): peon/PeonReady1 (\"Ready to work?\"), peon/PeonWhat1 (\"Yes?\"), ...\nglados (GLaDOS): glados/OhItsYou (\"Oh, it's you.\"), ..."
    }
  ]
}
```

See [MCP Resources](/api/mcp-resources) for catalog format details.

### Implementation Details

- **Queue-based playback** â€” sounds are queued and played sequentially
- **Non-blocking** â€” tool returns immediately (audio plays async)
- **Sound Effects device routing** â€” on macOS, routes to Sound Effects device if `use_sound_effects_device` is enabled
- **No-repeat logic not enforced** â€” MCP server plays exactly what the agent requests
