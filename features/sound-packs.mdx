## Managing Packs

### List Installed Packs

```bash
peon packs list
```

**Output:**
```
Installed packs:
  peon (active)
  peasant
  sc_kerrigan
  sc_battlecruiser
  glados
```

### Browse Registry

```bash
peon packs list --registry
```

Shows all 165+ packs available in the [OpenPeon registry](https://github.com/PeonPing/registry).

### Install Packs

<Tabs>
  <Tab title="Install Specific Packs">
    ```bash
    peon packs install glados,murloc,duke_nukem
    ```
    
    Comma-separated list of pack names.
  </Tab>
  
  <Tab title="Install All Packs">
    ```bash
    peon packs install --all
    ```
    
    Downloads all 165+ packs from the registry. Takes a few minutes.
  </Tab>
  
  <Tab title="Install During Setup">
    ```bash
    # Installer flags
    curl -fsSL https://peonping.com/install | bash -s -- --all
    curl -fsSL https://peonping.com/install | bash -s -- --packs=peon,glados,murloc
    ```
    
    Specify packs at install time.
  </Tab>
  
  <Tab title="Install Local Pack">
    ```bash
    peon packs install-local /path/to/custom-pack
    ```
    
    Install a custom pack from a local directory. Pack must contain `openpeon.json`.
  </Tab>
</Tabs>

### Switch Packs

```bash
# Switch to a specific pack
peon packs use glados

# Switch and install if not already installed
peon packs use --install murloc

# Cycle to next pack in rotation
peon packs next
```

**Effect:** Immediately changes the active pack. Next event uses the new pack's sounds.

### Remove Packs

```bash
peon packs remove glados,murloc
```

Deletes the specified packs from disk. Cannot remove the currently active pack.

## Pack Rotation

Rotate between multiple packs automatically with each session:

### Rotation Modes

<Tabs>
  <Tab title="random (Default)">
    Picks a random pack from the rotation list each session.
    
    **Configuration:**
    ```json
    {
      "pack_rotation": ["peon", "glados", "sc_kerrigan"],
      "pack_rotation_mode": "random"
    }
    ```
    
    **Set via CLI:**
    ```bash
    peon rotation random
    ```
  </Tab>
  
  <Tab title="round-robin">
    Cycles through rotation list in order.
    
    **Configuration:**
    ```json
    {
      "pack_rotation": ["peon", "peasant", "glados"],
      "pack_rotation_mode": "round-robin"
    }
    ```
    
    Session 1: peon → Session 2: peasant → Session 3: glados → Session 1: peon...
    
    **Set via CLI:**
    ```bash
    peon rotation round-robin
    ```
  </Tab>
  
  <Tab title="session_override">
    Manually assign a pack to each session using slash commands.
    
    **Configuration:**
    ```json
    {
      "pack_rotation_mode": "session_override"
    }
    ```
    
    **In Claude Code:**
    ```
    /peon-ping-use glados
    ```
    
    **Legacy alias:** `agentskill` mode (same behavior)
    
    **Expiration:** Old assignments expire after 7 days (configurable via `session_ttl_days`)
  </Tab>
</Tabs>

### Check Current Mode

```bash
peon rotation
```

**Output:**
```
Pack rotation mode: random
Rotation list: peon, glados, sc_kerrigan
```

## Directory-Based Pack Rules

Automatically switch packs based on project directory:

### Path Rules

```json
{
  "path_rules": [
    { "pattern": "*/work/client-a/*", "pack": "glados" },
    { "pattern": "*/personal/*", "pack": "peon" },
    { "pattern": "*/services/*", "pack": "sc_battlecruiser" }
  ]
}
```

**Matching:**
- First matching rule wins
- Supports `*` and `?` wildcards
- Overrides `pack_rotation` and `default_pack`
- Overridden by `session_override` assignments

### Bind Pack to Directory

```bash
# Bind current directory
cd ~/work/client-a
peon packs bind glados

# Bind with pattern
peon packs bind --pattern "*/services" sc_battlecruiser

# Remove binding for current directory
peon packs unbind

# List all bindings
peon packs bindings
```

**Output of `peon packs bindings`:**
```
Directory bindings:
  */work/client-a/* → glados
  */personal/* → peon
  */services/* → sc_battlecruiser
```

## Pack Priority Hierarchy

When multiple rules apply, PeonPing uses this priority:

1. **Session override** (`/peon-ping-use <pack>`)
2. **Path rules** (directory pattern matching)
3. **Pack rotation** (if `pack_rotation` is non-empty)
4. **Default pack** (`default_pack` config key)

**Fallback behavior:** If a pack is invalid or missing, PeonPing falls back through the hierarchy until it finds a valid pack.

## Creating Custom Packs

Build your own sound pack:

1. **Read the CESP spec:** [openpeon.com/spec](https://openpeon.com/spec)
2. **Follow the creation guide:** [openpeon.com/create](https://openpeon.com/create)
3. **See contribution docs:** [CONTRIBUTING.md](https://github.com/PeonPing/peon-ping/blob/main/CONTRIBUTING.md)

### Pack Structure

```
my-pack/
├── openpeon.json          # CESP manifest
├── session.start/
│   ├── greeting1.wav
│   └── greeting2.wav
├── task.complete/
│   ├── done1.wav
│   └── done2.wav
└── task.error/
    └── error1.wav
```

### Install Your Pack

```bash
peon packs install-local ~/my-pack
peon packs use my-pack
```
